# Railway 환경 변수 설정 가이드: bbb-analyzer

이 문서는 `bbb-analyzer` 프로젝트를 Railway에 배포할 때 필요한 환경 변수 설정 방법을 상세하게 설명합니다. 환경 변수는 애플리케이션의 동작 방식을 제어하고 중요한 보안 정보를 저장하는 데 사용됩니다.

## 1. 환경 변수란 무엇인가요?

환경 변수는 운영 체제나 호스팅 환경(이 경우 Railway)에서 애플리케이션에 전달하는 설정 값입니다. 코드 내에 직접 하드코딩하지 않고 외부에서 주입함으로써 보안을 강화하고 환경(개발, 테스트, 프로덕션)에 따라 설정을 쉽게 변경할 수 있습니다.

## 2. bbb-analyzer 필수 환경 변수

`bbb-analyzer` 프로젝트를 정상적으로 실행하기 위해 다음 세 가지 환경 변수가 반드시 필요합니다.

### 2.1. `NODE_ENV`

-   **설명**: 애플리케이션의 실행 환경을 지정합니다.
-   **값**: `production`
-   **이유**: 이 값을 `production`으로 설정하면 Express 서버와 React 프론트엔드가 프로덕션 모드로 최적화되어 실행됩니다. 불필요한 디버깅 정보가 숨겨지고 성능이 향상됩니다.

### 2.2. `JWT_SECRET`

-   **설명**: JSON Web Token(JWT)을 서명하고 검증하는 데 사용되는 비밀 키입니다. 사용자 인증 세션을 관리하는 데 필수적입니다.
-   **값**: 임의의 강력하고 유니크한 문자열 (예: `your_secret_key_here`)
-   **이유**: 이 키가 유출되면 악의적인 사용자가 임의로 인증 토큰을 생성하여 시스템에 접근할 수 있습니다. 따라서 **반드시** 추측하기 어려운 복잡한 문자열로 설정해야 합니다.
-   **생성 방법**: 터미널에서 `openssl rand -base64 32` 명령어를 실행하여 생성된 값을 사용하는 것을 권장합니다.

### 2.3. `DATABASE_URL`

-   **설명**: 애플리케이션이 연결할 데이터베이스의 주소와 인증 정보를 포함하는 연결 문자열입니다.
-   **값**: Manus에서 제공하는 TiDB Cloud (MySQL) 연결 문자열
-   **형식**: `mysql://[사용자명]:[비밀번호]@[호스트 주소]:[포트]/[데이터베이스명]?ssl={"rejectUnauthorized":true}`
-   **이유**: `bbb-analyzer`는 사용자 정보 등을 저장하기 위해 데이터베이스를 사용합니다. 이 변수가 올바르게 설정되지 않으면 애플리케이션이 데이터베이스에 연결할 수 없어 정상적으로 작동하지 않습니다.
-   **주의사항**: 연결 문자열 끝에 `?ssl={"rejectUnauthorized":true}` 옵션이 반드시 포함되어야 안전한 연결이 보장됩니다.

## 3. Railway에서 환경 변수 설정하기

1.  Railway 대시보드에서 `bbb-analyzer` 프로젝트를 선택합니다.
2.  좌측 메뉴에서 `Variables` 탭을 클릭합니다.
3.  `New Variable` 버튼을 클릭합니다.
4.  `Name` 필드에 환경 변수 이름(예: `NODE_ENV`)을 입력합니다.
5.  `Value` 필드에 해당 환경 변수의 값(예: `production`)을 입력합니다.
6.  `Add` 버튼을 클릭하여 환경 변수를 저장합니다.
7.  위 과정을 반복하여 `JWT_SECRET`과 `DATABASE_URL`도 추가합니다.

환경 변수를 추가하거나 수정하면 Railway가 자동으로 애플리케이션을 재배포하여 변경 사항을 적용합니다.
